# PRD: 출시 전 마케팅 장치 정리

## 1. 배경

현재 서비스는 관심 종목을 검색하면 최근 2주간 주식 유튜브 영상에서 해당 종목을 언급한 유튜버 의견을 긍정·신중·부정과 근거 영상으로 정리해준다.

MVP 기능은 출시 가능한 수준이지만, 현재 화면은 기능 설명 중심이라 출시 초반 유입과 공유를 만들 장치가 부족하다. 특히 프로젝트명인 `지금사도될까요?`와 실제 화면 브랜드인 `이 종목 어때?`가 분리되어 있어 사용자 기억에 남는 메시지가 약해질 수 있다.

출시 전에는 대규모 기능 추가보다 브랜드 통일, 첫 화면 훅 강화, 검색 결과 공유 장치를 우선 적용한다.

## 2. 목표

- 첫 방문자가 5초 안에 서비스 가치를 이해한다.
- 검색 전에도 볼거리가 있어 첫 화면 이탈을 줄인다.
- 검색 결과가 외부로 공유될 수 있는 최소 루프를 만든다.
- 출시 후 어떤 종목과 메시지가 반응을 얻는지 측정할 수 있게 한다.

## 3. 핵심 포지셔닝

### 서비스명

`지금사도될까요?`

### 한 줄 설명

사기 전에, 최근 주식 유튜버들이 이 종목을 어떻게 말했는지 먼저 확인하는 서비스.

### 메인 카피

```text
지금 사도 될까요?

사기 전에, 최근 2주 동안 주식 유튜버들이
이 종목을 어떻게 말했는지 먼저 확인하세요.
```

### 보조 카피

```text
긍정·신중·부정 의견과 근거 영상을 한눈에 정리해드립니다.
```

## 4. 출시 전 필수 범위

### P0. 브랜드 및 메타 정보 통일

현재 `stock-search.html`의 title, OG title, 브랜드명이 `이 종목 어때?`로 되어 있다. 출시 전에는 `지금사도될까요?`로 통일한다.

#### 요구사항

- 브라우저 title을 `지금사도될까요?`로 변경한다.
- OG title을 `지금사도될까요?`로 변경한다.
- 메인 브랜드명을 `지금사도될까요?`로 변경한다.
- meta description과 OG description을 새 포지셔닝에 맞게 변경한다.
- 기존 물음표 심볼은 유지 가능하다.

#### 권장 문구

```html
<title>지금사도될까요?</title>
<meta name="description" content="사기 전에, 최근 주식 유튜버들이 이 종목을 어떻게 말했는지 긍정·신중·부정 의견과 근거 영상으로 확인하세요.">
<meta property="og:title" content="지금사도될까요?">
<meta property="og:description" content="최근 2주 주식 유튜버 의견을 모아 종목별 분위기를 한눈에 보여드립니다.">
```

#### 완료 기준

- 첫 화면, 브라우저 탭, 공유 미리보기에서 동일한 서비스명이 보인다.
- `이 종목 어때?` 문구는 화면에서 제거되거나 보조 문구로만 사용된다.

### P0. 첫 화면 훅 강화

기존 `최근 주식 유튜브 언급 TOP 10`은 유용하지만 다소 기능적이다. 출시용 첫 화면에서는 사용자가 눌러보고 싶게 만드는 콘텐츠형 표현이 필요하다.

#### 요구사항

- 랭킹 영역 제목을 `오늘 유튜브에서 뜨거운 종목` 또는 `최근 유튜브에서 많이 언급된 종목`으로 변경한다.
- 랭킹 각 항목에는 가능한 경우 다음 정보를 표시한다.
  - 종목명
  - 언급 유튜버 수
  - 주가 등락률
  - 의견 분위기 요약
- 의견 분위기 요약 데이터가 없으면 기존 `N명 언급`만 표시해도 된다.

#### 권장 라벨

```text
오늘 유튜브에서 뜨거운 종목
긍정 우세
신중 우세
부정 우세
의견 갈림
언급 증가
```

#### 완료 기준

- 사용자는 검색하지 않아도 첫 화면에서 눌러볼 종목을 발견할 수 있다.
- 국내주식/미국주식 탭 구조는 유지한다.
- 기존 랭킹 API가 실패해도 fallback 랭킹이 정상 표시된다.

### P0. 검색 결과 공유 장치

출시 초반 유입을 만들기 위해 검색 결과를 쉽게 공유할 수 있어야 한다.

#### 요구사항

- 검색 결과 요약 영역에 `공유하기` 버튼을 추가한다.
- Web Share API를 지원하는 환경에서는 네이티브 공유창을 연다.
- Web Share API를 지원하지 않으면 공유 문구를 클립보드에 복사한다.
- 공유 성공 시 짧은 피드백 문구를 보여준다.

#### 공유 문구 예시

```text
삼성전자, 최근 2주 주식 유튜버 의견

긍정 3명 · 신중 2명 · 부정 1명
지금사도될까요?에서 근거 영상과 함께 보기
https://stockzip.kr/?q=삼성전자
```

#### URL 요구사항

- 검색 실행 시 query parameter를 갱신한다.
  - 예: `/?q=삼성전자`
- `q` query parameter가 있는 상태로 진입하면 해당 종목 검색을 자동 실행한다.
- 자동 실행이 부담되면 1차 출시에서는 검색창에 값만 채우고 사용자가 버튼을 누르게 해도 된다.

#### 완료 기준

- 모바일 브라우저에서 공유 버튼이 정상 작동한다.
- 데스크톱에서 클립보드 복사가 정상 작동한다.
- 공유 링크로 들어왔을 때 동일 종목을 다시 확인할 수 있다.

## 5. 출시 후 바로 볼 지표

기존 analytics 구조를 활용한다.

### 필수 이벤트

- `page_view`: 방문
- `search_submit`: 검색 실행
- `search_result`: 검색 결과 성공/실패
- `stock_detail_view`: 결과 상세 조회
- `source_click`: 원본 영상 클릭
- `share_click`: 공유 버튼 클릭
- `share_success`: 공유 또는 복사 성공

### 핵심 지표

- 방문자 대비 검색 실행률
- 검색 성공률
- 검색 결과당 원본 영상 클릭률
- 검색 결과당 공유 클릭률
- 공유 링크 유입 비율
- 많이 검색된 종목 TOP 20
- 결과가 없었던 검색어 TOP 20

## 6. 비범위

이번 출시 전 범위에서는 다음 작업을 하지 않는다.

- 회원가입
- 관심종목 저장
- 푸시 알림
- 결제
- 커뮤니티 기능
- 추천 알고리즘 고도화
- 대규모 UI 리디자인
- 백엔드 파이프라인 전면 개편

## 7. 사용자 시나리오

### 시나리오 1. 검색 유입 사용자

1. 사용자가 `지금사도될까요?`에 접속한다.
2. 메인 카피를 보고 관심 종목을 검색한다.
3. 최근 2주 유튜버 의견 요약을 확인한다.
4. 근거가 궁금한 의견의 원본 영상을 클릭한다.
5. 결과가 흥미로우면 공유 버튼으로 지인에게 보낸다.

### 시나리오 2. 랭킹 탐색 사용자

1. 사용자가 첫 화면에서 `오늘 유튜브에서 뜨거운 종목`을 본다.
2. 관심 종목 또는 낯익은 종목을 누른다.
3. 해당 종목 검색 결과로 이동한다.
4. 의견 분위기와 원본 영상을 확인한다.

### 시나리오 3. 공유 링크 유입 사용자

1. 사용자가 카톡, 커뮤니티, SNS에서 공유 링크를 누른다.
2. 해당 종목 검색 화면 또는 검색 준비 상태로 진입한다.
3. 결과를 확인한 뒤 다른 종목을 추가 검색한다.

## 8. 화면별 변경 요약

### 첫 화면

- 브랜드명: `지금사도될까요?`
- H1: `지금 사도 될까요?`
- 리드 문구: `사기 전에, 최근 2주 동안 주식 유튜버들이 이 종목을 어떻게 말했는지 먼저 확인하세요.`
- 검색 placeholder: `예: 삼성전자, 엔비디아, 반도체 ETF`
- CTA: `의견 확인하기`
- 랭킹 제목: `오늘 유튜브에서 뜨거운 종목`

### 검색 결과 화면

- 요약 영역에 긍정·신중·부정 카운트 유지
- 요약 영역 우측 또는 하단에 `공유하기` 버튼 추가
- 각 의견 카드의 원본 영상 링크 유지
- 하단 투자 권유 아님 고지 유지

## 9. 우선순위

### 1차 출시 전

- 브랜드명/메타 정보 통일
- 메인 카피 교체
- 랭킹 제목 교체
- 공유 버튼 추가
- 공유 이벤트 로깅

### 1차 출시 후 1주 내

- 공유 링크 자동 검색
- 의견 분위기 라벨 고도화
- 결과 없음 검색어 기반 추천 문구 개선
- 운영 대시보드에서 공유 지표 노출

### 반응 확인 후

- `오늘의 유튜브 종목 온도` 별도 콘텐츠 카드화
- 종목별 고정 공유 이미지 생성
- 관심종목 알림 또는 재방문 장치 검토

## 11. QA 기준

- 모바일 Safari/Chrome에서 첫 화면 문구가 접히거나 겹치지 않는다.
- 검색창 sticky 동작이 기존과 동일하다.
- 자동완성 선택 후 검색이 기존처럼 동작한다.
- 랭킹 종목 클릭 후 검색이 기존처럼 동작한다.
- 검색 결과가 없는 경우에도 공유 버튼은 노출하지 않거나 오류 없이 비활성 처리된다.
- 원본 영상 링크가 새 탭에서 열린다.
- 서버 미실행 상태의 안내 문구가 기존처럼 표시된다.
- analytics API 실패가 사용자 경험을 막지 않는다.

## 12. 출시 판단 기준

아래 조건을 만족하면 소프트 런칭을 진행한다.

- 첫 방문자가 서비스명을 기억할 수 있다.
- 검색 전에도 클릭할 종목 랭킹이 보인다.
- 검색 결과를 외부에 공유할 수 있다.
- 최소 이벤트 로그로 검색률과 공유율을 확인할 수 있다.
- 투자 권유가 아니라는 고지가 유지된다.
