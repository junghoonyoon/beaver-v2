# 비버 v2 작업본

종목 검색 앱 전용 작업본이다.

## 현재 제품 방향

`종목검색실행.command`를 실행하고 종목명을 검색하면 최근 14일 자막에서 해당
종목을 언급한 영상을 찾는다. 유튜버별 대표 영상 1개를 로컬 AI로 주문형 분석해
긍정·신중·부정 의견과 근거, 원본 링크를 보여준다.

검색창은 토스처럼 부분 입력 추천을 지원한다. `설정.txt`에 `공공데이터키`를
넣으면 공공데이터포털 KRX 상장종목정보를 하루 1회 받아와 국내 상장종목 전체를
자동완성 기준으로 쓴다. 예를 들어 `삼성`을 입력하면 KRX에 등록된 삼성 계열
상장종목이 함께 뜨고, 후보를 누르면 그 종목으로 바로 검색한다.

미국 종목은 별도 API 키 없이 Nasdaq Trader의 공개 Symbol Directory를 하루 1회
받아온다. `인텔`, `INTC`, `Intel`처럼 한글 별칭·티커·영문명 검색을 함께 지원한다.

검색하지 않은 종목은 AI 분석하지 않는다. 첫 검색은 상위 유튜버 5명만 우선
분석하고, 분석이 끝난 유튜버 카드부터 화면에 바로 추가한다. 이미 분석 캐시가
있는 추가 유튜버 의견은 함께 보여준다. 자막과 종목별 분석 결과는 캐시되어
같은 검색을 다시 할 때 빠르게 재사용한다.

## v2에서 달라진 점

- 영상 전체 시장 판정과 종목별 의견을 분리한다.
- 종목 의견은 `긍정·중립·부정·단순언급`으로 분석한다.
- 단순언급과 시황·거시·배분 채널은 종목 투표에서 제외한다.
- 같은 채널은 같은 종목에 최근 의견 1표만 갖는다.
- 종목 의견이 1명이면 `단일 의견`, 2명이면 `의견 참고`, 3명부터 우세 방향을 표시한다.
- 화면 문구를 `긍정 우세·신중 우세·부정 우세·의견 갈림`으로 완화했다.
- 영상 ID별 자막을 영구 캐시해 같은 영상은 다시 요청하지 않는다.
- 자막 실패도 기본 12시간 기억해 실행할 때마다 같은 실패를 반복하지 않는다.
- 자막 순서는 `캐시 → 직접 만든 자막 → InnerTube → Supadata → 무료 라이브러리`다.
- 같은 영상·같은 자막의 Gemini 분석 결과도 재사용한다.
- 채널 ID에서 업로드 재생목록 ID를 계산해 불필요한 YouTube API 호출을 줄였다.
- 영상 분석은 로컬 `qwen3:14b`를 먼저 사용하고, 로컬 출력이 실패하거나 JSON
  검증을 통과하지 못한 경우에만 Gemini를 fallback으로 사용한다.
- 전체 채널의 최근 영상 후보를 조회수로 정렬하고 상위 10개만 자막·AI 분석한다.
- 종목 검색은 기본 `검색분석수 = 5`로 제한해 첫 검색 대기시간을 줄인다.

캐시는 `pipeline/cache/`에 생성되며 Git에는 포함되지 않는다. 권한을 확보한 영상의
자막을 로컬 Whisper 등으로 직접 만들었다면
`pipeline/cache/manual_transcripts/영상ID.txt`에 넣으면 네트워크 자막보다 먼저 쓴다.

## 폴더 구조

| 경로 | 역할 |
|---|---|
| `앱화면/stock-search.html` | 종목 검색 화면 |
| `pipeline/` | 자막 수집, 종목 검색, 분석 서버 |
| `터미널없이_실행관리/` | 백그라운드 자동실행 등록/해제 |
| `pipeline/cache/` | 자막·검색·분석·KRX·미국 종목 캐시 |

## 실제 실행 준비

1. `로컬AI_준비.command`를 실행해 Ollama와 `qwen3:14b`를 준비한다.
2. `설정.example.txt`를 `설정.txt`로 복사하고 YouTube 키를 입력한다.
3. KRX 전체 종목 자동완성을 쓰려면 공공데이터포털 키를 `공공데이터키`에 입력한다.
4. Gemini 키는 로컬 실패 시 fallback이 필요할 때만 입력한다.
5. `종목검색실행.command`를 실행한다.

`종목검색실행.command`는 Vercel portless를 사용해 서버를 `stockzip` 이름으로 띄운다.
브라우저에서는 포트 번호 대신 아래 주소를 열면 된다.

```text
https://stockzip.localhost
```

수동으로 서버를 띄울 때도 같은 이름을 쓴다.

```bash
cd beaver-v2/pipeline
SEARCH_HOST=127.0.0.1 npx -y portless stockzip -- ./.venv/bin/python search_server.py
```

portless는 Node.js 24 이상이 필요하다. 처음 실행할 때 HTTPS 프록시와 로컬 인증서
설정 때문에 macOS 권한 확인이 뜰 수 있다.
권한 확인을 피해야 하는 임시 실행 환경에서는 `PORTLESS_PORT=1355`를 앞에 붙이면
`https://stockzip.localhost:1355`로 확인할 수 있다.

KRX 상장종목정보 캐시는 `pipeline/cache/krx_listed_stocks.json`에 저장된다. 서버가
켜질 때 오늘 갱신한 적이 없으면 백그라운드에서 자동 갱신한다. 즉시 갱신하려면:

```bash
cd pipeline
./.venv/bin/python sync_krx_listed.py
```

미국 상장종목정보 캐시는 `pipeline/cache/us_listed_stocks.json`에 저장된다. 별도
키는 필요 없다. 즉시 갱신하려면:

```bash
cd pipeline
./.venv/bin/python sync_us_listed.py
```

모바일에서 프로토타입을 보려면 `모바일미리보기.command`를 실행한다. 이 경로는
같은 와이파이 접속을 위해 기존 고정 포트를 그대로 쓴다. 터미널에
`http://맥IP:8765` 형태의 주소가 뜨면, 같은 와이파이에 연결된 휴대폰 브라우저에서
그 주소를 열면 된다.

터미널 없이 백그라운드 자동실행으로 쓰고 싶으면 `터미널없이_실행관리/` 폴더의
`터미널없이_자동실행_등록.command`를 한 번 실행한다. 해제도 같은 폴더의
`터미널없이_자동실행_해제.command`에서 관리한다.

설정의 `분석방식` 값:

- `local-first`: 로컬 우선, 실패 시 Gemini
- `ollama`: 로컬만 사용하며 Gemini를 전혀 호출하지 않음
- `gemini`: Gemini만 사용
- `openrouter`: OpenRouter의 OpenAI 호환 API만 사용

속도가 더 중요하면 `설정.txt`에서 `검색분석수 = 3`으로 낮추고, 의견 수가 더
중요하면 7~10으로 올리면 된다.

실행기가 `설정.txt`를 찾지 못하면 자동으로 예시 파일을 만들고 편집기로 연다.
필수 키가 비어 있거나 실행 단계가 실패하면 성공 메시지를 출력하지 않고 중단한다.
